# 原子笔记 → Obsidian 转换工具

将 **vivo 原子笔记**的备份数据一键转换为 **Obsidian** 可直接打开的 Markdown 文件夹结构。

---

## 功能特点

- **完整转换**：笔记内容（HTML）→ Markdown，保留段落、加粗、斜体、列表、标题、表格等格式
- **图片还原**：自动提取嵌入图片，生成 Obsidian `![[]]` 内链格式
- **文件夹映射**：原子笔记的文件夹结构完整还原，未分类笔记统一归入「未分类」文件夹
- **时间戳还原**：每条笔记的创建时间和修改时间与原始记录一致
- **数据安全**：以只读模式访问数据库，绝不修改原始备份文件；桌面旧结果自动移入回收站而非永久删除
- **开箱即用**：提供单文件 `.exe`，无需安装 Python 或任何依赖

---

## 使用方法

### 方式一：直接运行 exe（推荐）

1. 前往 [Releases](../../releases) 页面下载 `原子笔记转Obsidian.exe`
2. 双击运行
3. 按提示依次输入两个路径（见下方「如何找到备份路径」）
4. 等待转换完成，结果自动生成在桌面的 **「obsidian转换结果」** 文件夹

### 方式二：直接运行 Python 脚本

环境要求：Python 3.9+，无需额外安装第三方库

```bash
python vivo_to_obsidian.py
```

## 如何找到 vivo 备份路径

原子笔记的备份数据由 **vivo 办公套件**同步生成，同步后默认保存在：

```
C:\Users\你的用户名\AppData\Roaming\pcsuite\
```

该目录下包含两个你需要的文件夹：

| 输入提示 | 对应路径示例 |
|----------|-------------|
| `data 文件夹路径` | `C:\Users\你的用户名\AppData\Roaming\pcsuite\data` |
| `database 文件夹路径` | `C:\Users\你的用户名\AppData\Roaming\pcsuite\database` |

> **找不到 AppData 文件夹？**  
> 在资源管理器地址栏直接输入 `%AppData%\pcsuite` 回车即可跳转。

## 输出结构

转换结果保存在桌面的 `obsidian转换结果` 文件夹，结构如下：

```
obsidian转换结果/
├── attachments/        ← 所有图片附件
│   ├── IMG_xxx.jpg
│   └── ...
├── 我的文件夹1/        ← 对应原子笔记中的文件夹
│   ├── 笔记标题.md
│   └── ...
├── 我的文件夹2/
│   └── ...
└── 未分类/             ← 未归入任何文件夹的笔记
    └── ...
```

在 Obsidian 中打开 `obsidian转换结果` 文件夹作为 Vault 即可。

## 关于笔记标题

| 情况 | 处理方式 |
|------|---------|
| 有标题 | 直接使用原标题 |
| 无标题 | 取正文首句前 8 个字符作为文件名 |
| 标题重复 | 自动加全角序号后缀，如 `笔记名（1）`、`笔记名（2）` |

## 常见问题

**Q：提示找不到 NoteSync.db？**  
A：请确认 vivo 办公套件已完成原子笔记同步，`database` 文件夹下应存在 `NoteSync.db` 文件。

**Q：图片没有显示？**  
A：请确认 `data` 文件夹路径正确，且 `data/Note/Resource` 子目录存在。

**Q：可以重复运行吗？**  
A：可以。每次运行前，桌面已有的 `obsidian转换结果` 文件夹会自动移入回收站，不会直接覆盖，旧数据可从回收站恢复。

**Q：会修改我的原子笔记备份数据吗？**  
A：不会。程序以只读模式打开数据库，不会对备份目录中的任何文件进行写入或修改。
